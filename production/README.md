# Zoop SecOps — Wazuh + IRIS Production Setup

Combined security monitoring (Wazuh) and incident response (DFIR-IRIS) stack.

## What runs here

| Service | URL | Purpose |
|---|---|---|
| Wazuh Dashboard | `https://<server-ip>` | Security monitoring, 39+ agents |
| IRIS | `https://<server-ip>:8443` | Incident response case management |
| Wazuh Indexer | Internal (port 9200) | Data storage |
| Wazuh Manager | Internal (ports 1514, 1515) | Agent management |

## Requirements

- Linux VM (Ubuntu 22.04 recommended)
- Minimum **8GB RAM**, 4 CPU cores, 50GB disk
- Docker installed (script handles this)
- Ports open: **443**, **8443**, **1514**, **1515**

## Setup (run once)

```bash
git clone https://github.com/adinzoop/wazuh-infra.git
cd wazuh-infra/production

# Edit passwords before running
cp .env.example .env
nano .env

# Run setup (generates certs, starts everything)
chmod +x scripts/setup.sh
./scripts/setup.sh
```

## Start / Stop

```bash
docker compose up -d      # start
docker compose down       # stop (data kept)
docker compose down -v    # stop + wipe all data
docker compose logs -f    # view logs
```

## Default credentials

**Wazuh:**
- URL: `https://<server-ip>`
- Username: `admin`
- Password: set in `.env` → `INDEXER_PASSWORD`

**IRIS:**
- URL: `https://<server-ip>:8443`
- Username: `administrator`
- Password: set in `.env` → `IRIS_ADM_PASSWORD`

## Wazuh → IRIS integration

After IRIS is running:
1. Login to IRIS → click your avatar → **My Profile** → copy **API Key**
2. Edit `.env` → set `IRIS_API_KEY=<your key>`
3. Restart: `docker compose restart wazuh-manager`

From now on, any Wazuh alert at level 7 or above automatically creates a case in IRIS.

## Enrolling agents (team Macs)

In Wazuh Dashboard → **Agents Management → Summary → Deploy new agent**
- Select macOS → Apple silicon
- Server address: `<your-server-ip>`
- Copy and run the install command on the team member's Mac

## Notes

- Certs are auto-generated by `setup.sh` and stored in `certs/` (gitignored)
- `.env` is gitignored — never commit it
- Wazuh password hashes in `internal_users.yml` are for `REDACTED_INDEXER_PASSWORD` — change via `.env`
